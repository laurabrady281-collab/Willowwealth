Google OAuth blocked by Google Console configuration (403)

Google is rejecting the sign-in request. This is not a code issue — it's a configuration issue in the Google Cloud Console.

Common causes:

  • OAuth consent screen is not published or is still in "Testing" mode without test users added
  • The OAuth client type is not set to "Web application"
  • The redirect URI in the Google Console does not exactly match the one used by this app
  • The Google Client ID or Client Secret is from a different project or has been regenerated
  • Required APIs (Google+ API or People API) are not enabled in the project
